1. Layout

Use white space to create a visual hierarchy on the page. Design for small screens first using a single column layout.

Optimise for different screen sizes, but don’t make assumptions about what devices people are using.

  • See an example of GOV.UK grid layout

Spacing

  • spacing between elements is usually 5px, 10px, 15px or multiples of 15px
  • gutters are 15px for smaller screens and 30px for larger screens

Grid unit proportions

  • introduce columns as the content requires it – base column ratios on halves, thirds or quarters of the page width
  • use media queries to set breakpoints for screen sizes – find these in the GOV.UK front end toolkit _conditionals.scss file

2. Typography

Use the GDS Transport Website font in Light and Bold. It's licensed for use on the GOV.UK domains only. For services publicly available on different domains, use an alternative typeface like Arial. You can find common font colours in the colour palette.

  • See an example of GOV.UK Typography
  • Read a blog post about developing the GOV.UK typeface

Headings

  • use GDS Transport Website Bold
  • use headings consistently to create a clear hierarchy

Body copy

  • use GDS Transport Website Light
  • avoid using bold and italics

3. Colour

Always use the GOV.UK colour palette.

Colour and accessibility

  • when using colour on text and functional elements (like buttons) make sure the contrast is high enough for those with visual impairments
  • we aim to meet the AA colour contrast ratio of 4.5:1 recommended by the W3C - test to meet this standard

Text, background and button colours

  • use Sass variables in case colour values need to be updated – find these in the GOV.UK front end toolkit colours.scss file
  • text should be set in black $text-colour with secondary text in grey $text-secondary, links should be set in blue $link-colour and buttons should have white text

4. Icons and images

Avoid unnecessary decoration - only use icons and images if there's a real user need.

  • See an example of image usage on GOV.UK

Icons

  • if icons are needed ensure they are clear, simple and accompanied by relevant text
  • don’t hide functionality under icons

Images

If images are needed they should be landscape, 3:2 aspect ratio.

6. Forms

Keep forms as simple as possible – only ask what's needed to run your service.

See an example of GOV.UK form layout

Form fields and labels

Labels should be aligned above form fields.

Form focus states

All elements use the yellow focus style as a highlight, as either a fill or 3px outline.

Click on the label "Full name" or inside the form field to show the focus state.

Focus

  • #FFBF47
  • $yellow

Spacing between form elements

Ensure there is sufficient spacing between form elements.

Hint and example text

  • use hint text for supporting contextual help
  • don’t use placeholder text in fields
  • hint text should sit above a form field
  • ensure hint text can be read correctly by screen readers

It'll be on your last payslip. For example JH 21 90 0A.

Dropdowns

Avoid using dropdowns - use radio buttons or checkboxes instead.

Radio buttons

  • use these to let users choose a single option from a list
  • for more than two options, radio buttons should be stacked
  • radio buttons with large hit areas are easier to select with a mouse or touch devices

or

Checkboxes

  • use these to select either on/off toggles or multiple selections
  • for more than two options, checkboxes should be stacked
  • checkboxes with large hit areas are easier to select with a mouse or touch devices

7. Buttons

Use buttons to move though a transaction. Don't use them to represent choices – use radio buttons, checkboxes or links instead.

Button colours

  • buttons use the use the Sass variable $button-colour, Hex #00823B
  • buttons use the Sass variable $yellow, Hex #FFBF47 as a focus colour, as an outline of 3px
  • buttons are created using a Sass button mixin – find this in the GOV.UK front end toolkit buttons.scss file

8. Errors and validation

Summarise validation errors in a box at the top of your page. Use jump links in the summary box to reach the errors in the form.

Ensure error messages make sense when read by screen readers.

Form validation

  • use a red background and border on the box to visually connect the errors and messages in the form
  • apply the same colour to the area of the form where an error has occurred
  • error copy should be specific to the question and validation should identify all errors
  • errors should not cause pre-filled fields to clear
